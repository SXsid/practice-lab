## The pattern behind the pattern

What you've been describing this whole time is called:

> **Dependency Injection** — create dependencies once, pass them explicitly, swap them freely.

Singleton and DI are **opposites in philosophy**:
```
Singleton  →  object finds its own dependency  (hidden)
DI         →  dependency handed to the object  (explicit)
```

Modern backend frameworks are built entirely on DI:

- **Spring** (Java) → `@Autowired`
- **FastAPI** (Python) → `Depends()`
- **Go** → manual wiring in `main.go`
